# 🌐 Web Development - Assignment Submission Guide

Welcome to the **Web Development Domain** of **IGNITE'26**! 🚀

This domain is designed to help you become a modern frontend developer by building projects using industry-standard technologies. Throughout the program, you'll learn everything from web fundamentals to building dynamic applications with React.

Each assignment is designed to strengthen your problem-solving skills, development workflow, and understanding of modern web technologies.

---

## 📁 Domain Structure

```text
Web-Dev/
├── Assignment-1/
│   ├── questions.md
│   └── [Your Submission Folder]
├── Assignment-2/
│   ├── questions.md
│   └── [Your Submission Folder]
└── ...
```

Each assignment folder contains a `questions.md` file with detailed instructions, requirements, and objectives.

---

## ✅ Submission Instructions

### Step 1: Fork the Repository

Before starting any assignment, fork the IGNITE'26 repository to your GitHub account.

1. Open the repository.
2. Click the **Fork** button in the top-right corner.
3. Wait for GitHub to create a copy under your account.

---

### Step 2: Clone Your Fork

```bash
git clone https://github.com/<your-username>/ignite-2.0/.git
cd ignite-2.0
```

---

### Step 3: Read the Assignment

Navigate to the assignment folder and carefully read the `questions.md` file.

The file contains:

* 📋 Assignment Requirements
* 🎯 Learning Objectives
* 📅 Submission Deadline
* 💡 Bonus Challenges (if any)

---

### Step 4: Create Your Submission Folder

Inside the assignment directory, create a folder using the following format:

```text
NAME_ROLLNO
```

Example:

```text
ABHINAV_23012
```

---

### Step 5: Add Your Solution

Place all your project files inside your submission folder.

Example:

```text
Assignment-1/
└── NAME_ROLLNO/
    ├── src/
    ├── public/
    ├── package.json
    ├── README.md
    └── ...
```

For assignments containing multiple questions:

```text
Assignment-1/
└── NAME_ROLLNO/
    ├── Question-1/
    ├── Question-2/
    └── README.md
```

---

### Step 6: Commit and Push Your Changes

```bash
git add .
git commit -m "Assignment X - Web Dev - Your Name"
git push origin main
```

---

### Step 7: Create a Pull Request

1. Open your forked repository on GitHub.
2. Click **Compare & Pull Request**.
3. Create a Pull Request to the original repository.

#### Pull Request Title

Use the following format:

```text
Assignment X - Web Dev - Your Name
```

## 📝 Pull Request Description

Use the following template while creating your Pull Request:

```
## 📌 Project Overview

Briefly describe what you built.

## 🛠️ Technologies Used

- React
- JavaScript
- CSS / Tailwind CSS
- React Router (if used)
- Any additional libraries

## 🚧 Challenges Faced

Mention any challenges or issues you encountered.

## 📸 Screenshots

### Desktop View

(Drag and drop screenshot here)

### Mobile View

(Drag and drop screenshot here)

### Additional Features (Optional)

(Drag and drop screenshot here)

## 🔗 Live Demo

(Add deployment link if available)
```

---

## 📸 Screenshots Requirement

Screenshots are **mandatory** for every submission.

Please attach screenshots directly in the **Pull Request Description** by dragging and dropping the images into the description box on GitHub.

Include:

* 🖥️ Desktop View
* 📱 Mobile/Responsive View (if applicable)
* ⚙️ Feature Demonstration (optional)

⚠️ Pull Requests without screenshots may be marked as incomplete and may not be reviewed.

---

## 📋 Submission Checklist

Before submitting, ensure that:

* ✅ Assignment requirements are completed
* ✅ Code runs without errors
* ✅ Files are inside your submission folder
* ✅ Folder name follows the required format
* ✅ Changes are committed and pushed
* ✅ Pull Request is created successfully
* ✅ Screenshots are attached in the PR description
* ✅ Code is clean and readable

---

## 💡 Best Practices

* Write clean and maintainable code.
* Use meaningful file and variable names.
* Keep components reusable and organized.
* Follow proper project structure.
* Test your application before submission.
* Make small and meaningful commits.
* Follow React best practices where applicable.
